Definition:
A thread is a flow of execution in a program (kind of like a virtual CPU).

Parallelly:
JVM allows application to have multiple threads running concurrently.
Each thread can execute parts of your code in parallel with main thread.

Priority:
Each thread has a priority.
Threads with higher priority are executed in preference compared to
threads with lower priority.

JVM threads continuation:
JVM continues to execute threads until either of the following occurs:
    1. The exit method of class Runtime has been called
    2. All user threads have died/terminated.

Main thread:
    When JVM starts up, there is thread which calls the main method.
    This thread is called "main" or "main thread".
    Its the thread that starts/begin a program.

Daemon thread/Daemon: is a low priority thread that runs in the background
to perform tasks such (Background Services) as 
garbage collection(the classic example),
logging, monitoring, or cache cleanup.


JVM terminates itself when all user threads (non-daemon threads) finish
their execution.
